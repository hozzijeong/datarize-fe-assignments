# Frontend 과제 제출시 요구사항 및 TODO 리스트

## 가격대별 구매 빈도 차트

```
한 달 동안 발생한 구매 데이터를 바탕으로, 각 가격대의 제품이 얼마나 많이 구매되었는지 보여주는 차트를 구현하세요.
가격대는 2만원 이하부터 10만원 이상까지 만원 단위로 구분됩니다.
차트는 바 차트 형태로 시각화되어야 합니다.
날짜를 선택해서 특정 기간을 조회할 수 있도록 구현해주세요.
```

- [ ] 바 차트를 구현한다
- [ ] 가격을 끊는 범위는 만원 단위로 구분된다
- [ ] 날짜 선택을 통해 특정 기간을 조회할 수 있도록 한다

## 가장 많이 구매한 고객 목록 및 검색 기능

```
한 달 동안 가장 많이 구매한 고객을 내림차순/오름차순으로 정렬하여 보여주는 목록을 구현하세요.
기본 정렬은 ID입니다.
각 고객의 ID, 이름, 총 구매 횟수, 총 구매 금액을 표시하세요.
고객의 이름을 통해서 검색 가능해야 합니다.
```

- [ ] 가장 많이 구매한 고객을 내림차순/오름차순으로 정렬하여 보여준다
- [ ] 기본 정렬은 ID로 한다
- [ ] 각 고객의 ID, 이름, 구매 횟수, 구매 금액을 표시한다
- [ ] 고객의 이름을 통해 검색 가능하도록 한다. (검색시 자동 필터링?)

## 고객 ID 기반 상세 기능

```
특정 고객 Row를 클릭하면 해당 고객의 상세 구매 내역을 표시할 수 있는 기능을 구현하세요.
검색 결과에는 해당 고객의 구매 날짜, 구매한 제품 목록, 각 제품의 가격, 상품 썸네일이 포함되어야 합니다.
```

- [ ]고객 Row 선택시 해당 고객의 상세 구매 내역을 확인할 수 있또록 한다
- [ ]검색 결과에는 고객의 구매 날짜, 구매 제품 목록, 제품가격, 제품 썸네일을 포함한다

---

한 화면에 전부 다 보여주는 것을 목표로 한다. 고객 구매 정보는 별도 페이지로 분리한다

이때 구매 날짜를 기준으로 컴포넌트를 나누고 구매 제품 목록, 제품 가격 썸네일등을 나타낸다.
